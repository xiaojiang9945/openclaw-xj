# REPO_MAINTENANCE_POLICY.md

本仓库在 OpenClaw 开源代码基础上构建，并额外启用“安全优先”维护规范。

## 最高优先级

1. 必须遵守 `SECURITY_POLICY.md`（优先级最高）。
2. 如任何操作与 `SECURITY_POLICY.md` 冲突，必须暂停并等待仓库所有者明确确认。

## 默认边界

- 默认只在仓库目录内操作。
- 默认仅允许本地修改、测试、本地提交。
- 默认禁止 push / PR / 远程发布（除非所有者明确授权）。

## 提交流程（必须）

1. 给出当天计划（3-5项）。
2. 实施改动并完成本地验证（build/test/lint，按项目实际能力）。
3. 提交前做敏感信息检查（staged 文件 + staged diff）。
4. 先展示待提交文件列表，等待所有者确认后再 commit。

## 敏感信息规则（零容忍）

- 严禁提交/输出密码、Token、API Key、Cookie、私钥、会话数据、个人隐私信息。
- 严禁提交 `.env`、密钥文件、认证缓存、个人数据文件。
- 一旦检测到疑似敏感信息：立即阻断提交，并先给出清理建议。

## 建议

- 启用本仓库的 git pre-commit hook（见 `.githooks/pre-commit`）。
- 在 CI 中复用同一套敏感扫描逻辑。
