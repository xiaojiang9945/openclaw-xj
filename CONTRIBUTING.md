# CONTRIBUTING

感谢使用此模板仓库。

## 维护原则

- 安全优先、最小改动、先验证后提交。
- 严格遵守 `SECURITY_POLICY.md`（最高优先级）。
- 提交前必须执行敏感信息检查。

## 日常维护流程

1. 先给出 3-5 项维护计划。
2. 在最小范围内完成修改。
3. 本地验证（按仓库能力执行）。
4. 提交前扫描 staged 文件与 diff。
5. 展示待提交文件清单，等待所有者确认后再 commit。

## 提交约定

- 一次 commit 只做一类事情。
- commit message 解释“为什么改”。
- 默认不做 push/PR，除非所有者明确授权。

## 禁止项（摘要）

- 禁止提交/输出密钥、Token、隐私数据。
- 禁止提交 `.env`、密钥文件、认证缓存、个人数据文件。
- 发现疑似敏感信息立即停止提交并先告警。
